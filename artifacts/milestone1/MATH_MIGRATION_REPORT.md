# Milestone 1.5 Math Migration Report

## Scope and authority

This report covers `Plans/Milestone 1/05_GOLDEN_TESTS_AND_MIGRATION_PLAN.md`.
The Plan 10 findings are authoritative: USD/ETH price spacing is not delta
spacing, the approved default stop is 0.15% of entry, and sizing/debt must
include costs and confirmed executions.

## Formula migration

| Concern | Superseded formula | Authoritative formula | Reason | Runtime consumers |
|---|---|---|---|---|
| Level boundaries | `strike_width / level_count`, implemented independently and ambiguously described | `StrategyMathEngine.build_levels` with explicit `LEVEL_COUNT`, `PRICE_STEP`, `EQUAL_OPTION_LOSS`, or true `DELTA_STEP` configuration | Declares units and valuation mode; true delta spacing requires a delta source | exact engine, simulated coordinator, live/demo coordinators |
| Stop geometry | `entry + 0.15 * price_step` | explicit `ENTRY_PERCENT: entry + entry * rate` or `PRICE_STEP_FRACTION: entry + price_step * fraction` | Restores the approved 0.15%-of-entry rule without deleting the separately named compatibility mode | all level adapters and coordinators |
| Baseline sizing | `option_budget / tp_distance` | `(zone_budget + configured_buffer) / net_tp_profit_per_unit`, then instrument quantization and coverage recalculation | Includes fees, signed funding, spread, slippage, limits, and rounding | exact, one-level, multi-level, simulated, and live coordinators |
| Recovery sizing | `(option_budget + debt) / tp_distance` | `(zone_budget + confirmed_debt + configured_buffer) / net_tp_profit_per_unit`, then quantization and risk checks | Recovery uses confirmed debt and does not claim exact coverage after rounding | exact and live same-level recovery paths |
| Demo probe budget | `option_quantity * tp_distance` copied in `interfaces/demo_runner.py` | `build_single_virtual_level`, which delegates valuation, spacing, and stop geometry to `StrategyMathEngine` | Removes the last interface-layer zone-budget copies | D4, D5, and D6 demo paths (not executed by this plan) |

## Independent numerical before/after example

Inputs: short strike 3000 USD/ETH, long strike 2900 USD/ETH, option quantity
0.1 ETH, five levels, and entry stop rate 0.0015.

- Both approved PRICE_STEP/no-cost implementations produce boundaries
  `3000, 2980, 2960, 2940, 2920, 2900`.
- The superseded 15%-of-step stop for the first 20 USD zone was
  `3000 + 20 * 0.15 = 3003`.
- The restored ENTRY_PERCENT stop is
  `3000 + 3000 * 0.0015 = 3004.5`.
- The first zone option-loss budget is `0.1 * 20 = 2 USD` under explicit
  terminal valuation.
- Cost-blind sizing was `2 / 20 = 0.1 ETH`.
- With entry/TP fee rates of 0.0002, stop fee rate 0.0006, and a 0.001 ETH
  quantity step, net TP is `18.804 USD/ETH`, raw quantity is
  `0.1063603488619442671771963412 ETH`, and submitted quantity is `0.107 ETH`.
  Expected net TP is `2.012028 USD`, projected net stop is `0.7385889 USD`,
  and overcoverage is `0.012028 USD`.

The arithmetic above is fixed in JSON fixtures and was not generated by the
production engine.

## Affected implementation files

- `domain/strategy_math/*`: authoritative contracts, valuation, spacing,
  stops, costs, quantization, sizing, and recovery.
- `core/virtual_levels.py`: sole compatibility adapter from core/runtime level
  objects to the math façade; the legacy generator and public entry point were
  removed.
- `core/hedge_engine.py`: exact engine consumes authoritative levels and sizing.
- `application/demo_strategy_runtime.py`: simulated coordinator consumes
  authoritative levels.
- `application/one_level_coordinator.py`, `multi_level_execution.py`,
  `live_strategy_coordinator.py`, and `domain/live_recovery.py`: active sizing
  callers consume the façade migration adapter.
- `interfaces/demo_runner.py`: D4/D5/D6 level construction now uses the
  authoritative single-level adapter.

## Affected tests and evidence

- `test_golden_fixtures.py`: fixture schema, mandatory example, cost-bearing

## Remaining legacy callers

None. Repository search found no runtime import or call to the removed
`LegacyPriceStepLevelGenerator` or `generate_virtual_levels` entry point. The
old outputs survive only as fixed `legacy_characterization` fixtures/tests and
this migration documentation.
